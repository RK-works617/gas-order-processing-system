# gas-order-processing-system
受け付けた注文内容をスプレッドシートに転記し、在庫を自動更新。購入価格を計算して注文者に注文内容確認メールを送信。在庫が僅少になった場合はオーナーにお知らせメールを送るシステム。

## 機能

・Googleフォームから注文受付
・注文内容をスプレッドシートへ自動転記
・商品マスタから単価を取得
・購入金額を自動計算
・在庫数を自動更新
・在庫僅少時にオーナーへ通知メール送信
・注文者へ受付確認メール送信
・売上管理表へ売上データ自動転記
・売上管理表から、日次・月次の売上集計を実行。

## 作成目的

GAS学習用に作った個人開発作品です。

## 学習ポイント

このシステムを作成する中で、特に以下の点を学習しました。

### トリガーの作成

当初は e.values を利用してフォーム回答を取得しようとしていましたが、スタンドアロンスクリプトのフォーム送信トリガーでは利用できず、イベントオブジェクトの構造を理解する必要がありました。

最終的に、

e.response
getItemResponses()
getResponse()

を利用して回答内容を取得する方法を習得しました。

### データ型の違いと処理方法

売上集計機能追加に当たって、表の売上日と集計対象日が一致する日（年月）について集計を実行するようにしましたが、Date型のデータはそのままだと時刻データが入っているので見た目は同じ日付でも異なる値として扱われる場合がある。文字列型に直して日付の一致をチェックする必要があるということを学びました。売上集計対象日（年月）を入れるセルも、既存データから作成した候補リストよりプルダウンで選択することによって手入力による入力誤りや表示形式の不一致が発生しないように工夫しました。
